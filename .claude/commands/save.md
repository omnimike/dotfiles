Review the current dotfiles changes and push them with a descriptive commit message.

Steps:
1. Run `git status` and `git diff` to see what has changed in the repo.
2. Analyze the changes and write a short, descriptive commit message summarizing what was added, changed, or removed (e.g. "add telescope keybindings to nvim config" or "update zsh aliases and tmux prefix key"). Do NOT use a generic message like "save".
3. Stage all changes with `git add .`
4. Commit with the generated message.
5. Push to origin.

If there are no changes to commit, just tell the user there's nothing to save.
